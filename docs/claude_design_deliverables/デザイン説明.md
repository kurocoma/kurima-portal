# アクセス解析取得・請求関連取得 画面デザイン説明

対象: `/access-analytics`（アクセス解析取得）・`/billing`（請求関連取得）
成果物:

- インタラクティブモック `アクセス解析・請求関連取得画面.dc.html`（appbarの新ナビ2項目で画面切替、タブ切替、「取得」ボタンで進捗→完了/失敗の動きを確認可能。Tweaksで「ログイン切れで失敗」パターンに切替可）
- Jinja2テンプレート案 `deliverables/access_analytics.html` / `deliverables/billing.html`（既存クラスのみで構成）

## 1. 視覚的階層（2.3への回答）

`inventory.html` と同一の縦の流れを踏襲し、新しい階層表現を作らない。

1. **タブ**（`.tabbar`）— 「どのモールか」を最初に決める。ページ見出し直下。
2. **主パネル**（`.panel.center`）— タイトル＋1行の説明＋工程プレビュー（`.flow-chip` ①〜④）＋フォーム＋大きな主ボタン（`.btn-cta`）。各タブの操作はこの1ボタンのみ。
3. **進捗**（既存 `#progress-panel`）— 実行後にパネル直下へ出現。フローチップも `.is-active`/`.is-done`/`.is-failed` で連動。
4. **結果** — パネルの下。`result-bar`（完了タグ＋取得時刻）→ 要点（ファイル/一覧）→ `.fold`（技術詳細）。

フォーム項目は工程チップの**下**・CTAの**上**に置く（「何が起こるか」→「条件」→「実行」の順で読める）。

## 2. 情報の優先順位（特に /billing の結果一覧）

初期表示は「帳票・状態・行数・ダウンロード」の4列に絞る。

- **状態**が最重要: `final`=「確定」（`.tag-ok`＋チェック）、`provisional`=「未確定・取得せず」（`.tag-need`＋`row-need` で行ごと薄い黄色）、`unknown`=「状態を確認できず」（`.tag-need`）、`NO_DATA`=「データなし」（無彩色 `.badge`）。開発者語（final/provisional）は画面に出さない。
- 未確定スキップ等は表の上の `.note.warn`「確認事項」に現場向け文言で先出しする（inventory の warnings と同型）。
- **SHA-256・ファイル名・締め日・保存先**は `.fold`「取得の詳細（技術情報）」へ。SHA-256 は「照合コード」と言い換え、先頭8桁のみ。
- BillPay は社名・金額・精算書番号を表示しない旨を表の下に1行明記（マスク方針の見える化）。

## 3. 余白・情報密度

- 既存 `.panel`（26/28px）・`.filecard`・`.tbl` の余白をそのまま使用。新しい余白値を導入しない。
- /billing Yahoo!の「帳票種別」は `.opt` を横並びチップ状に3つ（1行）、「確定済みのみ取得」は説明文つき `.opt` 1枚を幅440pxに制限して中央配置——チェック4つを縦に積まない。
- 楽天アクセス解析の結果3件は `.filecard` を縦積み（行として読める）、Yahoo!全体分析4件は `.file-grid`（2×2）。

## 4. レスポンシブ

既存の860px/560pxブレークポイントで自然に崩れない構成のみを使用:

- `.file-grid` は860pxで1列化（既存CSS）。
- `.tbl` は `.tbl-scroll`＋`.tbl-min-560` で横スクロール退避。
- 560pxでは `.btn-cta` が全幅化（既存CSS）。`.field-row` の日付・selectは折返し。
- 帳票種別チェックの横並びは `flex-wrap:wrap` なので狭幅で自動折返し。

## 5. 状態表示

- 進捗中/完了/失敗/中止: 既存 `#progress-panel`＋`.flow-chip` の仕組みをそのまま使用。新規表現なし。
- `NEEDS_LOGIN` 系: `progress-error`「ログインの有効期限が切れています」＋ `progress-hint`「専用ブラウザでもう一度ログインしてから、取得ボタンを押し直してください。取得済みのデータが消えることはありません。」（error_hints.py の辞書と対応）。
- 連打対策: 既存 `.btn-cta:disabled` を踏襲（モックでも実行中は無効化）。

## 6. 新規クラス追加の要否

**追加不要**。ただし以下は既存パターン（dashboard.html と同様のインラインstyle）で補っており、頻出するようなら将来クラス化を提案:

- フォーム行の中央寄せ `style="justify-content:center"`（panel.center 内の field-row）
- 帳票種別チェックの横並びコンテナ（flex＋gap）
- BillPay結果表下のマスク方針の注記（12px muted）

## 7. その他の申し送り

- appbar のナビが10項目になり、1040px幅でも窮屈。`base.html` 側で「アクセス解析」「請求関連」への短縮を検討推奨（本依頼のスコープ外のため未変更）。
- 進捗の `stepChips` はタブごとにチップ数が異なるため、テンプレート側でタブ別に出し分ける形にした（progress.js の変更は不要）。
- `/preview` の遅延ロード応答は、実装時に `_access_analytics_results.html` / `_billing_results.html` への fragment 分離を推奨（inventory と同型）。テンプレート2枚制約との調整はレビューで合意のこと。
